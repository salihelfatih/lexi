# Lexi User Testing Kit

This folder is intentionally compact for contributors.

## What To Use

1. `testing-playbook.md` - one consolidated guide for setup, facilitation, consent, scoring, and debrief.
2. `sample-lease-scenarios.md` - 5 Ontario lease scenarios for anonymized test files.
3. `private-beta-checklist.md` - go/no-go checklist before each testing wave.
4. `05-recruitment-tracker.csv` - participant sourcing and scheduling.
5. `06-participant-log.csv` - aggregate session outcomes.
6. `07-analysis-summary-template.md` - one-page synthesis output.

## Suggested Flow

1. Recruit and schedule participants with `05-recruitment-tracker.csv`.
2. Assign each participant a scenario from `sample-lease-scenarios.md`.
3. Clear `private-beta-checklist.md` before the testing wave.
4. Run each session with `testing-playbook.md`.
5. Log outcomes in `06-participant-log.csv`.
6. Summarize findings using `07-analysis-summary-template.md`.

## Source of Truth

- Session goals and thresholds: `docs/product/user-testing.md`
- Priority context: `docs/product/roadmap.md`

## Data Handling

- Use participant IDs, not names.
- Keep notes free of personal identifiers.
- Delete uploaded documents after each session.
